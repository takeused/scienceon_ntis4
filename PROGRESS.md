## 최근 완료: 연구–IP 전환 공백 분석 고도화

분석 명칭을 `연구–IP 전환 공백 분석`으로 정리했다. 결과는 사업화 성공확률이 아니라 후속 검토 우선순위다.

### 분석 흐름

1. Cerebras는 특허 공백의 확정 판정자가 아니라 테마·검증 가설·검색어 변형 생성기로 사용한다.
2. 후보마다 한글 동의어·띄어쓰기 변형을 받고, 선택된 canonical 검색식을 논문과 특허에 동일 적용한다.
3. 공백 매력도, 전환·실행 근거, 데이터 신뢰도를 분리한다.
4. 정식 순위는 논문 20건 이상, 논문·특허 핵심 조회 정상, 신뢰도 60점 이상 후보만 대상으로 한다.
5. 논문 5~19건이면서 성장·신뢰도 조건을 만족하면 정식 순위가 아니라 `초기 탐색 후보`로 분리한다.
6. Top 3는 같은 AI 테마 또는 단어가 많이 겹치는 후보를 중복 선택하지 않는다.

### 점수 산식

- 공백 매력도 = 연구 기반 35% + IP 공백 신호 40% + 연구 모멘텀·안정성 25%
- IP 공백 신호 = 직접 로그 특허강도 결핍 70% + 신뢰 가능한 후보군 상대 신호 30%
- 내부 우선순위 = 공백 매력도 × 데이터 신뢰도
- 특허 0건은 검색 실패와 실제 공백을 구분할 수 없어 공백 신호를 만점으로 처리하지 않는다.
- 전환·실행 근거(NTIS·보고서·특허 전환 신호)는 우선순위와 분리해 후속 검토 방향으로 제시한다.

5. 로컬 브라우저 자동 연결 문제로 실제 화면 클릭 검증은 완료하지 못했다. 스크립트 로딩 순서 회귀 테스트와 문법 검사는 통과했다.

## 알려진 한계 및 다음 작업 우선순위

1. 현재 `후보군 대비 IP 공백` 기준선은 한 번의 분석에서 AI가 만든 후보들의 중앙값이다. 산업 전체 기준으로 만들려면 IPC/CPC 분류, 특허 패밀리, 권리 상태 데이터를 연동해야 한다.
2. 특허 건수만으로 FTO·유효권리·청구항 범위·시장성을 판단할 수 없다. 최종 투자 또는 사업화 결정 전 별도 검토가 필요하다.
3. NTIS 조회는 현재 프록시 경로가 필요할 수 있다. 브라우저 직접 실행 환경에서 NTIS 결과가 오류이면 프록시 실행 상태와 CORS를 먼저 확인한다.
4. 배포 전 최우선 조치: `BROWSER_API_MODE`를 끄고 브라우저에 있는 모든 API 키를 제거한 뒤, 서버 환경변수와 프록시 경로로 전환한다.
5. 실제 API를 사용한 브라우저 통합 검증은 아직 별도로 수행하지 않았다. 다음 작업 시 대표 키워드 2~3개로 결과·오류 상태·Top 3 다양성을 확인한다.

## 다음 고도화 후보: 산업 기준 IP 공백

구현은 가능하지만 두 단계로 나누어야 한다.

1. **IPC 기반 1차 고도화**: 현재 ScienceON 특허 응답의 `IPC`와 `PatentStatus`를 이용해 후보별 IPC 분포를 만들고, 같은 IPC·연도 집단의 논문 대비 특허 전환 수준과 비교한다. 이는 새 API 키 없이 먼저 구현할 수 있으나, ScienceON 검색 범위 안의 기준선이다.
2. **권리 품질 기반 2차 고도화**: KIPRISPlus를 서버 프록시로 연동해 국내출원번호 기준 특허 패밀리, 법적 상태 이력, 등록·소멸 상태를 결합한다. 이 단계에는 KIPRISPlus 이용 신청/API 키, 호출량·라이선스 확인, 브라우저가 아닌 서버 측 키 보관이 필요하다.

CPC는 현재 확인된 ScienceON·KIPRISPlus 메타데이터만으로 확정할 수 없으므로, 실제 공급 소스와 사용 조건을 별도로 선택·검증한 뒤 추가한다. 새 지표를 도입할 때는 현 `후보군 중앙값`과 `IPC·연도 기준선`을 동시에 표시해 기준선이 바뀌어도 결과를 추적할 수 있게 한다.
