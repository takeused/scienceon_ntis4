# UI 디자인 점검 기록

## 2026-07-13

- 검색·분석·연구비 화면의 색상, 테두리, 모서리, 그림자, 포커스 표현을 공통 CSS 토큰으로 정리했다.
- 기본 캔버스는 밝은 중성 배경으로 통일하고, 인디고 계열은 선택·강조 상태에만 사용하도록 했다.
- 헤더, 검색창, DB 전환, 탭, ScienceON/NTIS 표, 분석 카드, 모달, 페이지네이션, 토스트에 동일한 radius/elevation/focus 규칙을 적용했다.
- 모바일 폭에서 히어로·검색창·표 컨테이너의 밀도를 낮추는 반응형 보정을 추가했다.
- 기존 동작을 보존하는 CSS 레이어로 적용했으며, 다음 단계는 실제 브라우저에서 홈/검색/분석/다크모드 상태를 시각 점검하는 것이다.

## 2026-07-13 연구비 결과 화면 밀도 개선

- 표본 신뢰도 안내를 인라인 주황색 스타일에서 공통 중성 표면(`budget-warning`)으로 변경해 결과 카드와 톤을 맞췄다.
- 연구비 결과 컨테이너, 요약 카드, 규모 전환 칩, KPI, 분포 차트, 근거 설명, 표의 패딩·간격을 축소해 스크롤을 줄였다.
- 모바일에서는 KPI를 2열로 배치하고 결과 컨테이너 여백을 추가 축소했다.
- 검증: `node --check js\\ui.js`, 관련 테스트 24개 통과.

## 2026-07-13 연구비 요약 카드 좌우 균형 개선

- 단일 세로 흐름이던 `budget-hero`를 2열 그리드로 변경했다.
- 좌측에는 제안 금액·신뢰도·근거 품질·표본을, 우측에는 분포 진단 문구를 배치해 넓은 화면의 빈 공간을 정보 영역으로 활용한다.
- 700px 이하에서는 기존처럼 1열로 자동 전환해 모바일 가독성을 유지한다.

## 2026-07-13 분포 편차 진단 줄바꿈 보정

- 진단 컬럼 최소 폭을 300px로 확보하고 `word-break: keep-all`을 적용했다.
- `다소 큼` 같은 강조 라벨은 `white-space: nowrap`으로 보호해 글자가 낱글자로 깨지지 않도록 했다.
