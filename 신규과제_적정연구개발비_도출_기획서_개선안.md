## 0. 문서 개요

본 기획서는 신규 국가R&D 과제의 적정 연구개발비를 객관적으로 산출하기 위한 자동화 기능의 설계를 다룬다. 기존 "NTIS 검색 상위 5건 평균" 방식의 한계를 분석하고, 통계적 보정 + AI 유사도 평가 + 정책 메타데이터 가중치를 결합한 개선 로직을 제안한다.

> 🔄 **v2 변경사항:** 연구내용(absContent) 기반 검색·비교를 제거하고, **과제명(projNm) 단독** 기반으로 검색 및 유사도 평가를 단순화하였다. 실운용 시 연구내용 필드의 데이터 품질·누락 문제로 검색결과가 도출되지 않는 문제를 해소하기 위함이다.

| 항목 | 내용 |
|------|------|
| 목적 | 신규과제 예산 편성 시 근거 기반(Evidence-based) 적정 연구비 범위 제시 |
| 핵심 산출물 | 연간 적정 연구비 범위(하한~상한) + 산출근거 요약 + 유사과제 리스트 |
| 데이터 소스 | NTIS 통합검색 API (국가과학기술지식정보서비스) |
| AI 엔진 | Cerebras / Claude API (키워드 추출 + 유사도 평가) |
| 적용 대상 | 기획연구 단계의 신규사업 예산 규모 사전검토 |

## 1. 기존 제안 로직 검토 (문제점 분석)

### 1.1 기존 방식 요약

신규과제명 및 연구내용 입력 → NTIS 키워드 검색 → 상위 5건 산술평균 → 적정 연구비 제시

### 1.2 식별된 문제점

| 구분 | 문제점 | 영향도 | 예시 |
|------|--------|--------|------|
| P1 | 과제 스케일 불일치 | 치명적 | 대형 인프라사업(수백억)과 기초연구(수천만원) 혼재 → 평균 왜곡 |
| P2 | 수행 기간 미고려 | 높음 | 1년 단기과제 vs 5년 장기과제의 총연구비 직접 비교 |
| P3 | 키워드 매칭 한계 | 높음 | 검색 상위 ≠ 범위(Scope) 유사 과제 |
| P4 | 사업 유형 미구분 | 중간 | 출연(연) 주도 vs 산학연 협력 과제 혼재 |
| P5 | 물가·시점 미반영 | 중간 | 2015년 과제와 2024년 과제의 금액 직접 비교 |
| P6 | 표본 수 부족 | 중간 | 5건만으로 통계적 대표성 확보 불가 |

> ⚠️ **WARNING**  
> 단순 상위 5개 산술평균은 이상치(Outlier) 1건에 의해 결과가 수십 배 왜곡될 수 있으며, 예산 심의 시 객관적 근거로 활용하기 어렵습니다.

## 2. 개선 로직 (5단계 하이브리드 파이프라인)

기존 3단계를 5단계로 확장하여 정책 메타데이터 필터링과 시점 보정을 추가하고, 각 단계의 역할을 명확히 분리한다.

### Step 1. AI 키워드 추출 및 후보 확보

사용자가 입력한 **신규과제명**을 AI에 전달하여 NTIS 검색에 최적화된 키워드 세트를 추출한다.

> 🔄 **v2 변경:** 입력에서 `연구내용` 필드를 제거하고, **과제명만**을 기반으로 키워드를 추출한다. 연구내용은 NTIS 데이터 내 누락·비표준화 비율이 높아 검색 결과 미도출의 주요 원인으로 식별되었다.

| 항목 | 설명 |
|------|------|
| 입력 | 신규과제명, 희망 수행기간, 사업유형(선택) |
| AI 처리 | 과제명에서 핵심 기술키워드 3~5개 + 유사 표현·약어 확장 추출 |
| NTIS 호출 | 추출 키워드로 `projNm` 필드 대상 복수 쿼리 실행, 최소 30~50건 후보 확보 |
| 핵심 개선 | 과제명 토큰 분리 + 동의어 확장으로 검색 커버리지 확대 |

> ✅ **개선 포인트 (기존 대비)**  
> 기존: 사용자 입력 키워드 직접 검색 → 매칭률 낮음  
> 개선: AI가 과제명에서 핵심어를 분리·확장하여 NTIS `projNm` 기반 검색 적중률을 높임

---

### Step 2. 정책 메타데이터 필터링 (신규 추가)

NTIS 과제 데이터에 포함된 정책 속성을 활용하여 비교 대상의 동질성을 확보한다. 이 단계는 기존 로직에 없던 핵심 보완 사항이다.

| 필터 기준 | NTIS 필드 | 적용 방식 | 해결 문제 |
|-----------|-----------|-----------|-----------|
| 연구개발단계 | rndPhase | 사용자 선택(기초/응용/개발) 또는 자동추론 | P1 (스케일 불일치) |
| 과제 수행기간 | prdEnd - prdStart | 사용자 입력 희망기간 ±1년 범위 필터 | P2 (기간 차이) |
| 사업유형 | bizSect | 출연(연)/산학연/지자체 등 유형 매칭 | P4 (유형 미구분) |
| 수행년도 | prdStart | 최근 5년 이내 과제만 포함 | P5 (시점 미반영) |
| 연구비 규모 범위 | 계산값 | 사용자가 예상하는 대략적 규모 밴드 선택(선택적) | P1 보완 |

*이 단계를 통해 후보 30~50건이 약 15~25건으로 축소되며, 남은 과제들은 신규과제와 유사한 "맥락"의 과제로 한정된다.*

---

### Step 3. 데이터 정규화 및 통계적 이상치 제거

필터링된 과제들의 연구비를 통일된 기준으로 변환하고, 통계적 방법으로 극단값을 제거한다.

#### 3-1. 연간 예산 정규화

- **1순위:** NTIS 당해연도연구비(fundThyr) 필드 직접 활용
- **2순위:** 총연구비(totFund) ÷ 수행년수(prdEnd - prdStart + 1) 계산
- **보정:** GDP 디플레이터 또는 연구개발비 물가지수 적용하여 기준년도(2025년) 가치로 환산 (선택적 고도화)

#### 3-2. IQR 기반 이상치 제거

사분위수 범위(IQR = Q3 - Q1)를 계산하고, Q1 - 1.5×IQR ~ Q3 + 1.5×IQR 범위를 벗어나는 과제를 이상치로 판정하여 제외한다.

- 기존안의 "상하위 10% 제거"는 고정 비율 절사이므로 데이터 분포 무시 → IQR이 더 견고(Robust)
- 남은 과제가 10건 미만이면 사용자에게 경고 후 필터 기준 완화를 제안

---

### Step 4. AI 기반 정밀 유사도 평가 및 Top-N 선정

통계적으로 정제된 과제 목록(약 10~20건)을 대상으로, AI가 **과제명 기반** 기술적 유사성과 규모 적절성을 종합 평가한다.

#### 4-1. AI 평가 프롬프트 구조 (개선)

> 🔄 **v2 변경:** 유사도 평가 입력을 `projNm`(과제명) 단독으로 통일한다. `absContent`(연구내용 초록)는 데이터 누락률이 높아 평가 기준에서 제외하며, 과제명에 내포된 연구목표·기술분야·대상 정보를 AI가 해석하여 평가에 활용한다.

| 평가 차원 | 가중치 | 평가 기준 |
|-----------|--------|-----------|
| 기술적 유사성 | 45% | 과제명에서 추론되는 연구 목표·핵심 기술·방법론의 일치도 |
| 규모·범위 적합성 | 30% | 과제명 기반 개발 범위(TRL 단계), 성과물 유형의 유사도 |
| 최신성·정책 부합 | 15% | 최근 과제 우선, 현행 정책 방향과의 부합도 |
| 사업유형 일치 | 10% | 출연(연)/산학연 등 수행 구조의 유사성 |

> ※ 예산 구조 합리성(비목 구성) 평가 차원은 absContent 의존도가 높아 v2에서 제외하고, 기술적 유사성 가중치를 상향 조정하였다.

#### 4-2. AI 출력 사양

- 선정 과제 수: Top-5 (기본) ~ Top-7 (후보 풍부 시)
- 각 과제별: 선정 사유(2~3문장), 유사도 점수(0~100), 연간 환산 연구비
- 종합 의견: 적정 연구비 범위 제시 시 고려사항 코멘트

---

### Step 5. 최종 적정 연구비 산출 및 근거 생성

AI가 선별한 Top-N 과제의 연간 연구비를 기반으로 최종 값을 도출한다.

#### 5-1. 산출 방식

| 산출 지표 | 계산 방법 | 용도 |
|-----------|-----------|------|
| 중앙값 (Median) | Top-N 과제 연간 연구비의 중앙값 | 적정 연구비 기준점 (Primary) |
| 가중 평균 | AI 유사도 점수를 가중치로 한 가중평균 | 유사도 반영 보정값 |
| 범위 (Range) | Q1 ~ Q3 (또는 Min~Max) | 제안 가능 연구비 밴드 |
| 신뢰 구간 | 평균 ± 1SD (표본 충분 시) | 변동성 감안 범위 |

> ✅ **핵심 개선: 단순 산술평균 → 복합 지표**  
> 기존안은 산술평균만 제시하였으나, 중앙값(이상치 영향 최소화) + 가중평균(유사도 반영) + 범위(의사결정 유연성)를 함께 제시하여 예산 심의 시 다각적 근거를 확보한다.

#### 5-2. 최종 출력 구성

- **적정 연간 연구비:** 중앙값 기준 (예: 1.8억 원/년)
- **제안 범위:** Q1~Q3 (예: 1.2억 ~ 2.3억 원/년)
- **총 연구비 환산:** 연간 × 희망 수행기간 (예: 5.4억 ~ 6.9억 원 / 3년)
- **산출 근거 요약:** AI 생성 텍스트 (유사과제 분석 결과 종합)
- **근거 과제 리스트:** 과제명, 수행기관, 연간연구비, 유사도점수, 선정사유

## 3. 기존안 대비 개선사항 요약

| 구분 | 기존안 (3단계) | 개선안 v1 (5단계) | 개선안 v2 (과제명 단독) | 기대효과 |
|------|---------------|------------------|------------------------|---------|
| 검색 규모 | 상위 5건 | 30~50건 확보 후 정제 | 동일 | 통계적 대표성 확보 |
| 검색 기준 | 키워드 직접 검색 | 과제명+연구내용 기반 AI 확장 | **과제명(projNm) 단독** | 검색결과 미도출 문제 해소 |
| 필터링 | 없음 | 정책 메타데이터 5개 기준 필터 | 동일 | 동질 과제 비교 보장 |
| 정규화 | 총연구비 직접 비교 | 연간 환산 + 물가 보정(선택) | 동일 | 공정한 비교 기반 |
| 이상치 처리 | 상하위 10% 절사 | IQR 기반 통계적 제거 | 동일 | 데이터 분포 반영 |
| AI 유사도 입력 | 없음 | 과제명 + 연구내용(초록) | **과제명만 사용** | 데이터 누락 의존도 제거 |
| 산출값 | 산술평균 1개 | 중앙값 + 가중평균 + 범위 + 신뢰구간 | 동일 | 다각적 근거 제시 |

## 4. 파이프라인 처리 흐름

```
사용자 입력: 과제명 + 희망 수행기간 + 사업유형(선택)
│
▼
[Step 1] AI 키워드 추출 (과제명 → projNm 검색어 확장) → NTIS 복수 쿼리 → 후보 30~50건
│
▼
[Step 2] 정책 메타데이터 필터 (연구단계/기간/유형/년도) → 15~25건
│
▼
[Step 3] 연간 예산 정규화 + IQR 이상치 제거 → 10~20건
│
▼
[Step 4] AI 4차원 유사도 평가 (과제명 기반) → Top-5~7건 선정
│
▼
[Step 5] 중앙값 + 가중평균 + 범위 산출 → 적정 연구비 결과
```

| calcBudgetRange() | 최종 예산 산출 | Top-N 과제 연구비 | 중앙값/가중평균/범위 |
| renderDashboard() | 결과 UI 렌더링 | 산출 결과 객체 | 대시보드 HTML |

### 5.3 오류 처리 및 엣지 케이스

| 상황 | 처리 방안 |
|------|-----------|
| NTIS 검색 결과 10건 미만 | 키워드 완화 재검색 자동 수행, 그래도 부족 시 사용자 알림 |
| 메타데이터 필터 후 5건 미만 | 필터 기준 단계적 완화 (년도 → 기간 → 유형 순) |
| IQR 제거 후 3건 미만 | IQR 배수를 1.5 → 2.0으로 완화하여 재계산 |
| AI 평가 실패 (API 오류) | 통계값만으로 결과 제시 + AI 분석 불가 안내 |
| 전체 과제 연구비 0원/누락 | 해당 과제 자동 제외, 유효 과제 수 표시 |
